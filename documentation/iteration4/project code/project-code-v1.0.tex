\section{Περιγραφή Δομής Κώδικας}
Έχουμε 3 κύρια είδη κλάσεων: Pages, Controllers, Objects και Components.

Τα Pages είναι φυσικές σελίδες που συνήθως κληρωνομούν την κλάση MainWindow που
δημιουργεί το παράθυρο που βλέπει ο χρήστης.

Ta Objects είναι κλάσεις που αποθηκεύουν μια οντότητα π.χ. πελάτης, αεροπλάνο, πιλότος, κλπ...
Αυτή η κλάση αποθηκεύει στοιχεία μόνο για αυτό το αντικείμενο και έχει μεθόδους που μπορούν να
επεξεργαστούν τα στοιχεία τους.

Μετά υπάρχουν τα controllers όπου "ελέγχουν" και "αποθηκεύουν τα Objects. Κάθε controller
περιέχει μια λίστα από τα objects που ελέγχουν. Μπορεί επίσης να αποθηκεύει και να διαχειρίζεται
objects που είναι αποθηκευμένα μέσα στο object που αποθηκεύει. Π.χ. διαχειρίζεται passenger που είναι αποθηκευμένο
στο FlightBooking που είναι αποθηκευμένο στο Flight. Άρα το flightController ελέγχει το passenger παρόλο που
δεν το αποθηκεύει έμμεσα. Το controller μας βοηθάει να έχουμε ένα μέρος όπου διαχειρίζονται τα objects
που μας βελτιώνει το reusability και μειώνει τα σφάλματα καθώς δεν έχουμε duplicate γραμμές κώδικα και
έχουμε να τεστάρουμε λιγότερα πράγρματα.

Επίσης υπάρχουν τα components. Αυτά είναι κλάσης που κληρονομούν java swing classes. Αυτά χρησιμοποιούνται
για να μπορείς να φτιάξεις ένα gui. Τώρα θα μου πείτε γιατί φτιάχνουμε δικά μας αν υπάρχουν ήδη? Ο λόγος για
αυτός είναι ότι φτιάχνοντας δικές μας κλάσης ξέρουμε ακριβώς πως δουλεύει και μπορούμε να κρύψουμε την
πολυπλοκότητα που έχει η βιβλιοθήκη swing. Επίσης σημαίνει ότι όλα τα gui elements που έχουμε θα μοίαζουν
μεταξύ τους και υπάρχει συνοχή μεταξύ των οθονών.
